# 商品管理（Product）設計案 ※提案・未実装

> 目的：現在の「教材管理」を、教材＋グッズ＋アフィリエイト＋PR＋外部リンクを扱える「**商品(Product)管理**」へ、後方互換を保ったまま拡張する。
> 方針：**後から作り直しにならない／商品数が増えても管理しやすい／教材と互換／外部サービス連携しやすい**。

---

## 1. 現状の構造（棚卸し）

- **データ**：`MATERIALS_SEED`（コード内・26件）を `.map(makeMaterial)` で正規化。出所は単一配列。
- **主な項目**：`id`(自動採番)／`title,subtitle,thumb`／`format`(譜面/記事/動画/レッスン)／`content`(分類)／`difficulty,genres,tags`／`price,published`（`free`は導出）／説明系／`scoreData`・`article.blocks`・`videoUrl`等。
- **販売/所有**：`buyMaterial → Pay.purchase → DBX.owned`（id単位で所有記録）。`isOwned(id)`。無料は即付与、有料はStripe前提の「準備中」。
- **開く**：`openMaterial`＝記事ビューア / 譜面ビューア / 動画。
- **絞り込み**：難易度×ジャンル×内容×形式（`storeFilter`）。ラベルは `DIFFS/GENRES/CONTENTS/FORMATS` に集中定義。
- **練習記録**：`DBX.practice` も**教材IDがキー**。
- **管理画面**：`/admin`（パスワード）→ 形式フォルダ別の教材一覧・検索・並べ替え・新規作成・編集。

### 現状の限界
- 「教材」前提（譜面/記事/動画）に最適化。物販・外部リンク・アフィリエイト・PRの概念がない。
- 販売＝アプリ内付与のみ（外部決済・外部発送の導線がない）。

---

## 2. 商品(Product)統一モデル（中核設計）

**`kind`（商品の大分類）で種別を判別する discriminated union** にする。共通項目＋種別ごとのブロック。

```js
{
  id,                       // 自動採番（現行のまま）
  kind,                     // 'material' | 'goods' | 'affiliate' | 'pr' | 'link'
  title, subtitle, thumb,
  images: [url, ...],       // 商品画像（任意・グッズ等）
  category,                 // 種別ごとのカテゴリーキー（後述）
  tags: [ ... ],
  price, currency,          // 表示価格（外部商品は参考価格）／'JPY'既定
  published,                // 公開/下書き
  desc, longDesc,

  // 販売・導線（kindで使い分け）
  sale: {
    mode,                   // 'internal'（アプリ内＝教材）| 'external'（外部サイトで完結）
    provider,               // 'suzuri' | 'amazon' | 'booth' | 'base' | 'other'
    url,                    // 外部の購入/紹介URL
    ctaLabel,               // ボタン文言（例「SUZURIで購入」「Amazonで見る」）
    disclosure,             // 'pr' | 'affiliate' | null   ← 表示義務（ステマ規制対応）
  },

  // 教材のときだけ（既存の中身をこの名前空間へ）
  material: {
    format, content, difficulty, genres, recBpm, targetBpm, period,
    learnGoal, learn[], goals[], songs, notes,
    scoreData / article{blocks} / videoUrl / pdfUrl / audioUrl
  }
}
```

### 種別ごとの挙動
| kind | 例 | 開く/買う | 所有/練習 | sale.mode |
|---|---|---|---|---|
| **material** | 譜面/記事/動画/レッスン | アプリ内ビューア（現行） | ◯（owned/practice） | internal |
| **goods** | Tシャツ/タオル/ステッカー/スティック（SUZURI等） | **外部サイトを開く** | ✕ | external |
| **affiliate** | Amazon商品 | 外部（アフィリンク）＋「広告」表示 | ✕ | external |
| **pr** | 企業PR案件 | 外部＋「PR」表示 | ✕ | external |
| **link** | 外部サイト紹介 | 外部リンク | ✕ | external |

→ **外部商品は「カタログ＋リンクアウト」**。受注・決済・製造・発送はSUZURI等の外部で完結（アプリは在庫・決済を持たない＝最小実装で堅牢）。

## 3. 後方互換戦略（"作り直しにならない"の核）

1. **`kind` 既定＝'material'**：未指定の既存26件は自動的に material 扱い（データ無改変）。
2. **`makeMaterial` は `makeProduct({kind:'material', ...})` の別名に**：既存シード配列のコードはそのまま動く。
3. **教材固有ロジックは `kind==='material'` でガード**（所有/練習/ビューア）。外部商品はリンクアウト分岐。
4. **既存IDは維持**（owned/practice のキー）。
5. 命名は内部的に Product へ寄せつつ、`Catalog`/関数名はエイリアスで温存（段階的に改名）。

→ 既存の教材・購入・練習データは**一切壊れない**。

---

## 4. 外部サービス連携（SUZURI / Amazon 等）

- **方式：リンクアウト型**（MVP）。商品に `sale.url`（SUZURI商品ページ等）＋ `provider` ＋ `ctaLabel` を持たせ、ボタンで外部を開く。API連携・在庫同期は不要。
- 将来：SUZURIのAPI/埋め込みで商品自動同期も可能（任意・後付けできる設計）。
- **表示義務（重要・法令）**：`sale.disclosure` でラベルを自動表示。
  - `affiliate` → 「**広告（Amazonアソシエイト）**」表記（Amazonアソシエイト規約）。
  - `pr` → 「**PR / 広告**」表記（**ステマ規制＝景品表示法・2023/10〜**。広告である旨の明示が必須）。
  - 自社グッズ販売を本格化する場合、**特定商取引法の表記**ページが別途必要（外部サービス側に委ねるなら、その旨明記）。

## 5. 管理画面（商品管理）

- **トップを種別フォルダに**：📚教材 / 👕自社グッズ / 🛒アフィリエイト / 📣PR / 🔗外部リンク（各に件数 計/公開/下書き）。
- 教材フォルダ内は現行どおり**形式（譜面/記事/動画/レッスン）**でサブ分け。グッズは**カテゴリー**でサブ分け。
- 検索・並べ替え・折りたたみは現行の `教材管理` ビューを**そのまま一般化**。
- 新規作成フォームは **最初に「種別」を選択 → 種別に応じた項目だけ表示**：
  - material → 現行フォーム＋📝譜面エディタ。
  - goods/affiliate/pr/link → タイトル/画像/価格/カテゴリー/**外部URL**/provider/CTA文言/**表示ラベル**。
- 自動採番ID・「保存＝更新コード生成→デプロイ」は現行どおり流用。

## 6. カテゴリー設計（種別ごと）

- 教材：現行 `CONTENTS`（基礎/フィル/グルーヴ…）。
- グッズ：新設 `GOODS_CATEGORIES`＝アパレル(apparel)/小物(accessory)/ドラム用品(drumware)/ステッカー(sticker)/その他。
- アフィリエイト/PR：機材(gear)/書籍(book)/アクセサリ… など共通の `SHOP_CATEGORIES`。
- ラベルは現行と同様**1箇所に集中定義**（重複管理しない）。

## 8. 重視点への対応まとめ

| ご要望 | 設計上の担保 |
|---|---|
| 後から作り直しにならない | `kind` discriminated union＋名前空間。新種別は項目追加で対応＝構造改変不要 |
| 商品数が増えても管理しやすい | 種別フォルダ＋件数＋検索/並べ替え、カテゴリーの集中定義 |
| 教材管理との互換 | kind既定=material・makeMaterial別名・既存ID維持＝既存データ無改変 |

## 10. 確定事項（ご回答反映）と実装状況

**ご回答**：①5種でOK ②グッズは **SUZURI＋BASE** ③種別フォルダOK ④**無料版＝広告(PR/アフィリ)表示／有料版＝広告なし** ⑤**1ストアに種別タブで統合**。

**フェーズ1（実装・デプロイ済み）**
- `makeProduct(kind)` ＋ `makeMaterial` をラッパ化（**既存26教材は完全に従来どおり**＝テスト確認済み）。
- 外部商品（goods/affiliate/pr/link）：カード/詳細/**外部サイトへリンクアウト**（`window.open`）。SUZURI/BASE/Amazon等の `provider` 対応。
- **広告の出し分け**：`affiliate`/`pr` は **無料版のみ表示・有料版は非表示**（`productVisibleToUser`）。表示時は**「広告/PR」ラベル自動表示**（ステマ規制対応）。
- **ストアに「種別」フィルタ追加**（すべて/教材/自社グッズ/アフィリエイト/外部リンク）。教材専用の絞り込み（難易度等）選択時は外部商品を除外。
- サンプル外部商品2件（Tシャツ=SUZURI／スティック=Amazonアフィリ）を仮で掲載（URL・公開状態は後で差し替え）。

**フェーズ1の残り（次の増分）**
- 管理画面：**種別フォルダ**表示（現在の `教材管理` は形式フォルダのみ＝外部商品が未表示）。
- 新規作成フォーム：**先頭で種別を選択→種別別の項目**（外部商品は 画像/価格/外部URL/provider/表示ラベル）。
- ユーザーストアの**種別タブ**化（現状はフィルタチップ。タブUIに整える）。
