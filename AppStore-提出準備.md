## 3. プライバシー申告（App Privacy / 栄養成分ラベル）

> ★最終申告は各SDKの公開情報で要確認：AdMob（Googleの「Apple App Privacy details」）・RevenueCat・Google Sign-In。以下は本アプリ構成に基づく推奨申告。

**トラッキング（App Tracking Transparency）**：**あり**
- AdMob（無料版広告）でIDFA等を第三者広告に利用 → **ATTダイアログを表示**（実装済み）。

**収集するデータ（推奨申告）**
| データ種別 | 例 | 用途 | 個人と紐付け | トラッキング |
|---|---|---|---|---|
| 識別子（Device ID / IDFA） | 広告識別子 | 第三者広告 | される場合あり | はい |
| 使用状況データ | 広告の操作 | 第三者広告/分析 | いいえ | はい |
| 購入 | サブスク状態 | App機能（RevenueCat） | いいえ（匿名App User ID） | いいえ |
| 粗い位置情報（※AdMob申告に準ずる） | 国/地域 | 広告 | いいえ | はい |

**収集しない／開発者が保持しないもの**
- 譜面データ（ユーザーコンテンツ）：端末内＋**ユーザー自身のGoogleドライブ（appdataフォルダ）**に保存。**当方サーバーには送信・収集しない**。
- 氏名・メール・連絡先：アプリ内で収集しない（お問い合わせは任意のメール）。
- ※Googleドライブ連携は「ストレージ連携」であり、当方がユーザーデータを収集するものではない。

**結論**：プライバシーラベルは主に**AdMob由来（トラッキングあり・識別子/使用状況/位置）**＋**RevenueCat由来（購入・非トラッキング）**。広告を入れる以上「トラッキングする」申告が必要。

## 7. 提出前チェックリスト（コード/設定）
- [ ] RevenueCat：`REVENUECAT_IOS_API_KEY` を実キーに／エンタイトルメント`pro`／商品ID一致／Sandbox購入・復元・トライアル確認
- [ ] AdMob：`ADMOB_APP_ID_IOS`/`ADMOB_BANNER_ID_IOS` を実IDに／Info.plist `GADApplicationIdentifier` 実IDに／`ADMOB_TESTING=false`／（任意）SKAdNetworkItems追加
- [ ] テスト用Proトグル非表示（`DEV_MODE=false`）／特別コードは審査メモのみで配布
- [ ] `/admin` 非搭載・UIに管理導線なし（iOSバンドルで確認）
- [ ] 外部決済導線なし（Stripe等の実体なし）
- [ ] 利用規約/プライバシー/特商法ページが公開URLで開ける（ペイウォール/設定からリンク）
- [ ] アプリアイコン1024px・スプラッシュ設定
- [ ] バージョン/ビルド番号設定（Xcode）／`ITSAppUsesNonExemptEncryption=NO`
- [ ] 実機で全機能の最終動作確認（縦横回転・手書き・PDF/画像保存・バックアップ/復元）

## 8. TestFlight → 申請（手順）
1. Xcodeで Archive（Product → Archive）→ Organizer → Distribute App → App Store Connect → Upload
2. App Store Connect → TestFlight でビルド処理完了を待つ（輸出コンプライアンス回答）
3. 内部テスター（自分）で配布・動作確認 →（任意）外部テスター
4. 「Appの提出準備」：メタデータ/スクショ/プライバシー/価格/IAP/審査メモ を埋める
5. **審査へ提出** → リジェクト時は理由に沿って修正（多くは3.1.2課金表記・プライバシー・デモアクセス）
