# 教材管理ガイド（整理案 / レビュー用）

> 目的：今後教材を追加する際に「テンプレートを埋めるだけ」で登録でき、データの重複なく管理できる状態にする。
> ※これは**整理案**です。確認後に、必要なコード反映（スキーマ正規化・詳細画面の項目追加など）を進めます。

---

## 1. 現状の棚卸し（いま教材はこう管理されている）

- 教材は `index.html` 内の単一配列 **`MATERIALS_SEED`**（約25件）が出所。ここを編集するだけで追加/編集/削除できる（全ユーザー共通）。
- 区分のラベルは1か所に集約済み：`DIFFS`（難易度）/`GENRES`（ジャンル）/`CONTENTS`（内容）/`MTYPES`（種類）。
- `learn`/`goals` は **`CONTENT_INFO`** にカテゴリ別の既定があり、教材側で省略すると自動補完（＝重複入力を減らす仕組みが既にある）。
- 開き方は `type==='article'`→記事ビューア、`scoreData`あり→譜面ビューア、それ以外→動画プレースホルダ。
- 決済は `PAYMENTS_LIVE=false`。有料(`free:false`)は「準備中」表示。`free:true`は即入手。

### 棚卸しで分かった「重複・あいまい」
1. **`type` が形式（score/article/video/lesson）と分類（basic/fill/groove）を混在** → 一番の整理ポイント。
2. **`free` は `price===0` から導出できる**（二重管理）。
3. **`author` は常に固定**（既定値にできる）。
4. **`period` が「練習期間」と「読み物/—」を兼ねる**（記事・動画では別項目が自然）。
5. ご要望の **サブタイトル・使用楽曲・注意点・タグ・PDF有無・練習音源有無** は未対応。

### 毎回入力している実質的な内容（追加のたびに書くもの）
`id / title / type / content / difficulty / genres / price / thumb` ＋ 説明系(`desc, longDesc, learnGoal, learn, goals`) ＋ 形式に応じて `scoreData` か `article.blocks` か `videoUrl`。

## 2. 推奨スキーマ（正規化案）

**狙い：形式と分類を分離し、冗長項目を既定値化し、不足項目を追加。**

### 必須（コア）
| 項目 | 例 | 説明 |
|---|---|---|
| `id` | `mat_16beat` | 一意キー（`mat_` + 英数。変更しない） |
| `title` | `16ビート基礎` | タイトル |
| `format` | `score` | **形式**：`score`/`article`/`video`/`lesson`（＝開き方。旧`type`の形式部分） |
| `category` | `groove` | **分類**：`CONTENTS`のキー（旧`content`を一本化） |
| `level` | `beginner` | 対象レベル（旧`difficulty`。値は据え置き） |
| `price` | `0` | 価格（0＝無料。`free`は廃止し導出） |

### 推奨（販売・表示で使う）
| 項目 | 例 | 説明 |
|---|---|---|
| `subtitle` | `〜オープンHHまで〜` | **新規**：タイトル下の補足 |
| `genres` | `['funk','pops']` | ジャンル |
| `thumb` | `🌀` | アイコン |
| `learnGoal` | … | 学習目的 |
| `summary` | … | 短い説明（旧`desc`） |
| `description` | … | 詳細説明（旧`longDesc`） |
| `learn[]` | […] | 習得内容（省略時カテゴリ既定） |
| `goals[]` | […] | 習得できること（省略時カテゴリ既定） |
| `songs[]` | `['課題曲A']` | **新規**：使用楽曲 |
| `notes` | … | **新規**：注意点 |
| `tags[]` | `['16分','ゴースト']` | **新規**：検索/分類タグ |
| `period` | `3週間` | 想定練習期間（練習系のみ） |
| `recBpm`/`targetBpm` | 60/100 | テンポ |
| `published` | `false` | 公開状態（有料の販売開始） |

### 形式ごとの中身（いずれか）
| 形式 | 項目 |
|---|---|
| `score` | `scoreData`（譜面JSON）, 任意で `audioUrl`(練習音源), `pdfUrl` |
| `article` | `article.blocks[]` |
| `video` | `videoUrl` |

### 既定値（factoryで自動補完＝入力不要）
- `author='DRUM STUDIO'` ／ `free = (price===0)` ／ `published`（無料は`true`、有料は明示） ／ `learn/goals`（省略時`CONTENT_INFO[category]`）

## 3. データ・フォルダ構成の整理案

単一ファイル(`index.html`)アプリのため「フォルダ」は概念整理として：

```
教材データの出どころ
└─ index.html
   ├─ ラベル定義（1か所）   : DIFFS / GENRES / CATEGORIES / FORMATS
   ├─ 既定値（1か所）       : CONTENT_INFO（カテゴリ別 learn/goals）
   ├─ makeMaterial()        : テンプレ→正規化（既定補完・検証）
   └─ MATERIALS_SEED[]      : カテゴリ別にセクション分割
        ├─ // ===== 基礎 =====
        ├─ // ===== グルーヴ =====
        ├─ // ===== フィル =====
        ├─ // ===== 記事 =====
        └─ // ===== 動画 =====
```

**Googleドライブ側（教材の素材＝譜面/音源/PDF/動画）**　※既存のコピーガイドと整合：
```
マイドライブ/
└ ドラム教材/
   ├ _登録テンプレート.md      ← 下記テンプレを置く
   ├ 基礎/  グルーヴ/  フィル/  記事/  動画/  楽曲/  採譜/
   └ （各カテゴリ）/(教材ID)_(教材名)/
        ├ 譜面データ.txt   ├ 練習音源.mp3   ├ 教材.pdf   └ メモ.md
```
→ **教材IDでフォルダ名を揃える**ことで「コード上のID＝ドライブのフォルダ」が一目で対応。

**重複削減の方針**：①`type`二役を`format`+`category`に分離 ②`free`は`price`から導出 ③`author`/`learn`/`goals`は既定補完 ④ラベルは既存の集中定義を継続。

## 実装状況

- ✅ **① `makeMaterial()` ファクトリ＋既定値**（実装・デプロイ済み）
  - 既定補完：`author='DRUM STUDIO'` ／ `genres=[]` ／ `free=(price===0)` ／ `published=(無料は公開・有料は準備中)`。
  - 既存25件は `.map(makeMaterial)` で正規化（**明示値はそのまま＝挙動は不変**）。新規は `price` だけ書けば `free/published` を導出。
- ✅ **② 詳細画面に サブタイトル／使用楽曲／注意点 を表示**（実装・デプロイ済み）
  - 追加項目：`subtitle`（タイトル下）／`songs`（配列＝箇条書き・文字列＝そのまま）／`notes`（注意点）／`tags`（任意）。
  - 例として `mat_song1`（課題曲：Pops Ballad）に3項目を投入済み（中身は自由に編集可）。
- ✅ **③ `type`→`format`+`content`(分類) の正規化**（実装・デプロイ済み・後方互換つき）
  - `format`：形式＝開き方（`score`/`article`/`video`/`lesson`）。**種類バッジ＝形式**（譜面教材→「譜面」）。`makeMaterial()` が旧 `type`・中身から自動導出（既存25件も自動で正しい形式に）。
  - `content`：分類＝詳細の「内容」（グルーヴ/基礎 等）。**種類＝形式・内容＝分類**で完全分離。
  - ストアの絞り込み「種類」も **形式（譜面/記事/動画/レッスン）** に変更。旧 `type` は後方互換で残置（消さなくてよい）。
- ⏳ **④ 登録テンプレートのドライブ配置**（4-1/4-2 をドライブの `_登録テンプレート.md` に）＝任意・お好みで

## 5. 管理者画面（/admin・静的分離）※実装・デプロイ済み

- **ユーザー画面（ `/` ）**：管理導線は一切なし。旧「タイトル5回タップ」は**廃止**。
- **管理者画面（ `https://drumscore.pages.dev/admin` ）**：開くと**パスワード**を要求。通すと管理者モード（左下に🔒バナー、「一般画面へ」で戻る）。
  - パスワードは `index.html` の `ADMIN_PASS`（既定 `drum-admin-2026`）。**必ず変更してください**。
  - ⚙設定 → 管理者メニューに **📚教材管理 / 📋教材用にコピー / ✏️新規教材を作成**。
  - **✏️新規教材を作成**：項目を入力 → 「登録用コードを生成」→ コピー → 私に渡す → コード追加してデプロイ＝公開。
- ⚠ 静的アプリのため、パスワードはソースに含まれます（簡易保護）。本格的な認証・即時公開が必要になったらバックエンド導入（前回の選択肢B）に移行できます。Googleログインの管理者メール判定への差し替えも可能です。
